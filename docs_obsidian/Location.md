# Location (локация)

Таблица: `locations` · Модель: `backend/app/models/location.py`

Точка на карте — привязка [[Quest|квеста]] к реальному или абстрактному месту. Одна локация может иметь несколько квестов.

## Связи

- **Квесты на метке:** [[Quest]] (`location_id`, опционально)
- **Без локации:** квест не отображается на карте (раздел «Бездомные задания» в UI)

## Поля

| Поле | Тип | Назначение |
|------|-----|------------|
| `name` | string(128) | Название места («Таверна у рынка», «Офис-крепость») |
| `description` | text | Описание локации |
| `place_type` | enum | RPG-тип места — см. ниже |
| `latitude` | decimal(9,6) | Широта (для карты) |
| `longitude` | decimal(9,6) | Долгота (для карты) |
| `address` | string(256) | Человекочитаемый адрес |

## Типы мест (`place_type`)

| Значение | RPG-смысл | Примеры в жизни |
|----------|-----------|-----------------|
| `settlement` | Поселение | Район города |
| `tavern` | Таверна | Кафе, место встреч |
| `wilderness` | Дикие земли | Парк, поездка |
| `dungeon` | Подземелье | Сложное / неприятное место |
| `stronghold` | Крепость | Дом, офис |
| `shrine` | Святилище | Спортзал, врач |
| `market` | Рынок | Магазин, ТЦ |
| `unknown` | Неизвестно | По умолчанию |

## Использование в квестах

- Квесты типа `exploration` чаще всего привязаны к [[Location|локации]]
- Квесты типа `bounty` могут иметь дедлайн **и** локацию (куда идти выполнять)
- Координаты опциональны — можно задать только `name` + `place_type` для абстрактных мест

## Запросы на карте

Индекс `location_id` на таблице `quests` позволяет быстро получить все активные квесты с меткой. См. [[Quest#Индексы]].
