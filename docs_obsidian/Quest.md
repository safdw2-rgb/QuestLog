# Quest (квест)

Таблица: `quests` · Модель: `backend/app/models/quest.py`

Центральная сущность квестового дневника. Каждый [[Quest]] принадлежит одному [[Adventurer|искателю]], может быть привязан к [[Faction|фракции]] и [[Location|локации]], содержит [[QuestStep|этапы]] и [[JournalEntry|записи дневника]].

## Связи

- **Владелец:** [[Adventurer]] (`adventurer_id`)
- **Категория:** [[Faction]] (`faction_id`, опционально)
- **Метка на карте:** [[Location]] (`location_id`, опционально)
- **Цепочка:** `parent_quest_id` → другой [[Quest]] (главный квест → подквесты)
- **Дочерние сущности:** [[QuestStep]], [[JournalEntry]]

## Поля

| Поле | Тип | Назначение |
|------|-----|------------|
| `title` | string(200) | Название квеста |
| `description` | text | Лор и контекст задания |
| `quest_type` | enum | Тип квеста — см. ниже |
| `status` | enum | Жизненный цикл — см. ниже |
| `difficulty` | enum | `trivial` → `legendary`, влияет на [[Economy\|награды]] |
| `xp_reward` | int | Плановый опыт при создании |
| `gold_reward` | int | Плановое золото при создании |
| `xp_earned` | int | Фактически начисленный опыт |
| `gold_earned` | int | Фактически начисленное золото |
| `deadline` | datetime | Дедлайн; обязателен для `bounty` |
| `started_at` | datetime | Начало квеста |
| `completed_at` | datetime | Момент завершения |
| `failed_at` | datetime | Момент провала |
| `fail_reason` | text | Причина провала — остаётся в дневнике |
| `sort_order` | int | Порядок в списке дневника |

## Типы квестов (`quest_type`)

| Значение | RPG-смысл | Особенности |
|----------|-----------|-------------|
| `main` | Главный сюжет | Высокие [[Economy\|награды]], может иметь дочерние [[Quest\|квесты]] |
| `side` | Побочное | Стандартный квест |
| `daily` | Ежедневный ритуал | Пересоздаётся по расписанию (логика в сервисе) |
| `bounty` | Контракт | Ожидается `deadline`; провал по истечении срока |
| `exploration` | Исследование | Новая территория, часто с [[Location\|локацией]] |
| `boss` | Босс-файт | Высокая сложность, триггер [[Achievement\|достижений]] |

## Статусы (`status`)

```
active ──► completed
   │
   ├──► failed
   ├──► deferred
   └──► abandoned
```

| Статус | Описание |
|--------|----------|
| `active` | Квест в работе |
| `completed` | Успешно завершён — начисляется [[Economy\|награда]] |
| `failed` | Провален — запись с `fail_reason`, обычно без награды |
| `deferred` | Отложен, но не провален |
| `abandoned` | Сознательно заброшен |

> Проваленные квесты **не удаляются** — остаются в дневнике с печатью «Провалено». См. [[JournalEntry]].

## Индексы

`adventurer_id`, `status`, `quest_type`, `faction_id`, `location_id`, `deadline` — для выборок «активные», «просроченные bounty», «квесты на карте».
