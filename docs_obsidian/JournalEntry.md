# JournalEntry (запись дневника)

Таблица: `journal_entries` · Модель: `backend/app/models/journal.py`

Свободный текст в квестовом дневнике: прогресс, итог, причина провала, лор от игрока.

## Связи

- **Автор:** [[Adventurer]] (`adventurer_id`)
- **Квест:** [[Quest]] (`quest_id`, опционально — запись может быть без привязки)

## Поля

| Поле | Тип | Назначение |
|------|-----|------------|
| `entry_type` | enum | Тип записи — см. ниже |
| `content` | text | Текст записи |
| `created_at` | datetime | Время создания |

## Типы записей (`entry_type`)

| Значение | Когда создаётся |
|----------|-----------------|
| `progress` | Заметка о ходе выполнения [[Quest\|квеста]] |
| `completion` | Итог при статусе `completed` |
| `failure` | Запись при провале — дополняет `fail_reason` в [[Quest]] |
| `lore` | Свободная заметка игрока |

## Проваленные квесты в дневнике

Когда [[Quest]] переходит в `failed`:
- Поле `fail_reason` фиксирует краткую причину
- [[JournalEntry]] типа `failure` может содержать развёрнутый рассказ
- Квест **остаётся** в дневнике с печатью «Провалено»
